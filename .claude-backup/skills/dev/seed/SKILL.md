# /seed — Scaffold a New Tracked Repo

You are creating a fully functional repo with an agent router, .claude/ infrastructure, and brain vault registration. Parse `$ARGUMENTS` for a repo name and domain description.

## Phase 1: Understand

Parse the arguments. The first word is the repo slug (lowercase, hyphenated). Everything after is the domain description.

Ask the user ONE AskUserQuestion with up to 4 questions:

1. **Top-level directories** — Suggest 3-5 domain-appropriate defaults inferred from the description. Example: a garden repo might get `plants/`, `logs/`, `resources/`. Ask the user to confirm or adjust.
2. **Common utterances** — "What will you say most often when working in this repo?" Give 4-6 examples inferred from the domain. These become the agent router intents. Example for garden: "log today's watering", "add a new plant", "what's in season".
3. **Existing files to import?** — A path to copy in, or "starting empty" (default).
4. **Target directory** — Default: the parent directory of `$BRAIN_VAULT_PATH` (i.e., the development root), plus `/{repo-name}`. Confirm or override.

## Phase 2: Confirm Router

Design 4-8 domain-specific IF/THEN router intents from the user's utterances and your domain understanding. Each intent follows this pattern:

```
**IF** user [does X] →
_"example phrase 1", "example phrase 2"_
→ [Action: work in directory, run command, etc.]
```

Always append these 2 universal intents at the end:

```
## Status & overview

**IF** user asks about status, what's here, or wants an overview →
_"what do I have", "show me everything", "status"_
→ Summarize repo contents using directory counts and recent files

## Direct work

**IF** user asks to do something not covered above →
→ Use best judgment, work in the most appropriate directory
```

Present the full router to the user for confirmation. Adjust if they give feedback.

## Phase 3: Generate

Always include `prompts/sprints/` and `prompts/context/` — these hold frontier-model prompt templates (generated by `/research-sprint`) organized by sprint number, and shared context documents.
